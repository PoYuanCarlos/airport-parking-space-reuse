Project Overview

From parking decline forecast to airport access strategy.

01

Transportation Problem

Airports may face reduced long-term parking demand as shared autonomous mobility becomes more common, while curbside pickup, drop-off, staging, and access-management pressure may increase.

02

Decision-Support Goal

The project translates uncertain mobility adoption scenarios into interpretable indicators, dashboard outputs, and practical space-reuse recommendations.

03

Portfolio Value

The case study demonstrates transportation-domain reasoning, P.A.C.E. workflow, EDA discipline, PMP-style delivery thinking, and stakeholder-oriented communication.

Decision Output Matrix

Turning analysis into stakeholder-ready recommendations.

Data Insight Decision Recommendation
Long-term parking demand by scenario High shared-mobility adoption may reduce parking utilization. Avoid overbuilding fixed parking capacity. Delay fixed expansion and evaluate flexible reuse options.
Curbside and staging demand proxies Parking decline may shift pressure to pickup/drop-off and staging zones. Reallocate selected space toward managed shared-mobility operations. Pilot geofenced pickup, staging, and curbside turnover controls.
Revenue sensitivity by access mode Parking revenue may decline while new access fees may emerge. Compare parking, staging, charging, commercial, and mobility-hub revenue options. Use a phased revenue replacement strategy.
Reuse-option scorecard Different reuse options perform differently by adoption scenario. Select options based on flexibility, revenue, operations, and equity. Use a decision matrix rather than one universal recommendation.
PMP Delivery Layer

Project structure, acceptance criteria, and public-release control.

Scope

Analyze airport parking-demand reallocation and space-reuse scenarios using public or synthetic data, then convert results into a dashboard and decision-support case study.

Acceptance Criteria

The dashboard must be interpretable, the data workflow must be explainable, recommendations must link to evidence, and public materials must avoid privacy or copyright risks.

Risks

Key risks include weak public data availability, overclaiming automation, misleading scenario assumptions, dashboard over-complexity, and public-release privacy concerns.

Communication Plan

GitHub Pages communicates the case study; Tableau communicates dashboard insights; README and SQL snippets support technical review; summary outputs support managers and recruiters.
